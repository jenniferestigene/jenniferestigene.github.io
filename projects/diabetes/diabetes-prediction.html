Project Overview

Can machine learning identify diabetic patients using diagnostic measurements? This analysis develops a classification model to predict diabetes onset while minimizing missed diagnoses.

The Challenge

Diabetes affects over 537 million adults globally, with early detection critical for effective treatment. This project addresses significant missing data (48.7% insulin, 29.6% skin thickness), class imbalance, and the need for interpretable models in medical contexts.

The Approach

Systematic evaluation of six machine learning algorithms on the PIMA Indians Diabetes Database, with comprehensive preprocessing, hyperparameter tuning, and emphasis on recall (catching true diabetic cases) over precision.

Key Findings

Logistic Regression Outperformed Complex Models

Despite being the simplest algorithm, Logistic Regression achieved the highest F1-score (0.661) and ROC-AUC (0.843), outperforming ensemble methods like Random Forest and Gradient Boosting.

Hyperparameter Tuning Critical for Medical Applications

Optimization improved recall from 52% to 72%, reducing missed diabetic cases from 26 to 15 in the test set—a 39% improvement in catching true positives.

Glucose as Dominant Predictor

Glucose level emerged as the strongest predictor (coefficient: 1.17), followed by BMI (0.52) and Age (0.41), aligning with established medical knowledge about diabetes risk factors.

Missing Data Patterns Informative

Created missingness indicators captured patterns in absent data. Median imputation preserved distributions while addressing 48.7% missing insulin values and 29.6% missing skin thickness measurements.

Favorable Trade-off for Screening

The tuned model accepts 5 additional false positives to catch 11 more true positives—appropriate for medical screening where missed diagnoses are costlier than additional testing.

Interpretability Enables Clinical Use

Logistic regression's transparent coefficients allow clinicians to understand which features drive predictions, unlike black-box ensemble methods with similar accuracy.

Methodology

A systematic machine learning pipeline combining data preprocessing, model selection, and optimization tailored for medical diagnostics.

1

Data Preprocessing

Converted biologically impossible zeros to NaN, created missingness indicators, applied median imputation, and standardized features using StandardScaler.

2

Algorithm Comparison

Evaluated six models using 5-fold stratified cross-validation: Logistic Regression, Random Forest, Gradient Boosting, SVM, KNN, and Decision Tree.

3

Performance Evaluation

Assessed models across accuracy, precision, recall, F1-score, and ROC-AUC. Prioritized recall to minimize false negatives in medical screening context.

4

Hyperparameter Optimization

GridSearchCV explored 48 parameter combinations for Logistic Regression, testing regularization strength, penalty type, solver, and class weights.

5

Feature Importance Analysis

Extracted and ranked feature coefficients to identify key predictors. Validated clinical alignment with established diabetes risk factors.

6

Error Analysis

Examined misclassification patterns and prediction probabilities to understand model limitations and identify borderline cases requiring additional evaluation.
